# Portfolio

Astro / SCSS / JavaScript で制作したポートフォリオサイトです。

実務のコーディング案件を想定し、静的サイトとしての表示速度、保守しやすいCSS設計、公開前の検証しやすさを意識して構成しています。

## 使用技術

- Astro
- SCSS
- JavaScript

## 設計方針

このサイトでは、厳密なFLOCSS準拠よりも、実務での修正しやすさと影響範囲の追いやすさを優先しています。

CSSは `l-` / `c-` / `p-` / `u-` のような命名を使いながら、汎用化しすぎない粒度で分けています。
大枠の配置やグリッドは `l-`、
再利用できる部品は `c-`、
サイト固有・ページ固有のまとまりは `p-`、
余白や背景などの単一目的の調整は `u-` に寄せています。

特に意識しているのは、汎用部品をページ都合で肥大化させないことです。
ボタン、見出し、アイコンなど複数箇所で使う見た目は `component` に置きますが、
ページ固有の配置や文脈に依存する表現は `project` 側に閉じ込めています。

例えばトップページでは `.p-home-*` を中心に、セクション単位・要素単位でスタイルを管理しています。
共通化できるものだけを抽象化し、固有の表現は固有の場所に残すことで、後から修正する時に影響範囲を読みやすくしています。

余白やgapなどの細かい調整は `utility` クラスとして用意し、SCSSのループやmixinで生成しています。
PC/SPで必要な差分も、なるべくmixinや変数を通して扱う方針です。

色、余白、フォントサイズ、ブレークポイントなどは `global` の変数やmixinを通して扱い、直値が増えすぎないようにしています。

## CSSレイヤー

- `foundation`: resetやbaseなど、全体の土台
- `global`: 色、フォントサイズ、mixin、ブレークポイントなどの共通定義
- `layout`: header、footer、section、gridなど大枠のレイアウト
- `component`: ボタン、見出し、テキスト、アイコンなど再利用する部品
- `project/common`: サイト共通だが、このサイト固有のまとまり
- `project/page`: ページ単位のスタイル
- `project/parts`: 複数ページで使うが、汎用componentまでは切り出さない固有パーツ
- `utility`: margin、padding、gap、背景色など単一目的の調整用クラス

## 公開範囲について

制作実績は公開用に匿名化しています。
社名、URL、実際のスクリーンショット、案件固有の情報は、必要に応じて汎用表現やダミー画像に差し替えています。

コードは、実装方針、CSS設計、コンポーネント構成を確認できる範囲に絞って公開しています。
